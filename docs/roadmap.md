# Roadmap

## v0.1: Protocol

Goal: define the open operating model.

Deliverables:

- repository documentation
- founder intent transcription
- project audit
- BOS kernel infrastructure specification
- positioning for owner-operators
- company management model
- open-source adoption strategy
- agent roles
- memory model
- artifact templates
- GitHub label taxonomy
- autonomy policy
- example configuration
- growth and revenue operating model
- business objective and revenue loop
- always-on autonomy protocol
- risk and compliance gate model
- easy setup and preset model
- owner command center model

No runtime required.

## v0.2: Local CLI

Goal: run BureauOS locally against one repository.

Deliverables:

- `bureau init`
- setup wizard
- presets
- `bureau intake`
- `bureau run`
- `bureau status`
- local config loader
- local artifact writer
- provider adapter interface

## v0.3: Owner Interface MVP

Goal: give the owner one place to see the agency.

Deliverables:

- local dashboard
- coordinator chat placeholder
- clients view
- projects view
- opportunities view
- approvals view
- reports view
- settings view

## v0.4: GitHub Integration

Goal: operate through GitHub issues and pull requests.

Deliverables:

- GitHub issue reader
- GitHub comment writer
- label router
- PR description generator
- check report parser
- artifact comments

## v0.5: Project Teams

Goal: support multiple projects with isolated memory.

Deliverables:

- project registry
- client registry
- project manager agents
- project memory store
- cross-project executive reports

## v0.6: Growth and Revenue

Goal: support visibility, marketing, conversion, sales, and client success.

Deliverables:

- brand memory
- offer memory
- content pipeline
- campaign briefs
- lead pipeline
- opportunity pipeline
- pricing briefs
- proposal briefs
- ad campaign briefs
- social and creative briefs
- compliance reviews
- conversion audits
- client account plans

## v0.7: Always-On Daemon

Goal: continuously observe, prioritize, dispatch, verify, report, and learn.

Deliverables:

- scheduler
- event loop
- event watcher
- signal classifier
- stale issue detection
- failing check detection
- autonomous triage runs
- daily operational report
- bounded retry system

## v0.8: Pull Request Execution

Goal: allow policy-bounded implementation runs.

Deliverables:

- branch creation
- code agent adapter
- test command runner
- PR opening
- review agent
- QA verification
- security gate

## v1.0: Autonomous Agency OS

Goal: manage multiple clients and projects with always-on teams.

Deliverables:

- supreme coordinator
- multiple project teams
- company, client, project, run, and decision memory
- GitHub-native work execution
- policy-bounded PR lifecycle
- release readiness workflow
- human escalation workflow
