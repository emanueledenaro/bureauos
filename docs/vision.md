# Vision

BureauOS is an open-source framework for running an autonomous AI software agency.

Most AI coding tools optimize for this flow:

```text
prompt -> code -> pull request
```

BureauOS is designed for a broader professional workflow:

```text
client input -> supreme coordinator -> project team -> product analysis
-> UX refinement -> technical planning -> implementation -> QA
-> security review -> code review -> release -> operational report
```

The goal is not to replace human ownership. The goal is to let an owner define policies and priorities once, then let an always-on AI organization operate inside those boundaries.

## What BureauOS Should Become

BureauOS should become the open operating model for autonomous AI software teams.

It should support:

- multiple clients
- multiple projects
- multiple internal teams
- isolated project memory
- always-on monitoring
- GitHub-native work execution
- agile ceremonies between AI agents
- policy-bounded autonomy
- human escalation when risk, ambiguity, or authority requires it

## What Makes It Different

BureauOS is not centered on one model provider.

The underlying model can be OpenAI, Anthropic, Google, local models, Codex, Claude Code, Gemini CLI, or any future agent runtime. BureauOS defines the organizational logic above those models.

```text
Model provider != agent role
Agent runtime != company operating model
```

The durable value is the agency protocol:

- who owns the conversation
- who owns a project
- who can act
- what memory is loaded
- what artifact is produced
- what policy allows the next action
- when a human must be asked

## Product Thesis

GitHub already hosts the software work.

BureauOS adds the missing executive layer:

```text
An autonomous AI software agency that plans, delegates, builds, reviews,
verifies, and reports through GitHub-native artifacts.
```

## Non-Goals

BureauOS should not start as:

- a generic multi-agent chat room
- an unbounded self-modifying agent
- a hidden SaaS-only system
- a prompt marketplace
- an automatic production deployment engine
- a tool that merges code without explicit policy

The first open-source version should be inspectable, conservative, and useful before it becomes powerful.
